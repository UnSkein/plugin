# 다오wsl 스킬 카탈로그

> 모리가 띄운 작업 다오(다오wsl)에 어떤 스킬이 있고 언제 쓰는지의 참조 정보. 모리 클라이언트가 작업을 의뢰할 때 정확한 스킬을 가리키기 위한 목록이다.
>
> **자동 생성 — 직접 고치지 않는다.** 단일 출처는 `dao-skills/.claude/skills/*/SKILL.md`(스킬별 앞머리)와 `dao-skills/CLAUDE.md`(단계 순서). 스킬을 추가·변경하면 `python3 bin/gen-dao-catalog.py`로 이 파일을 다시 만든다.

## 단계 순서 (1 -> 7)

다오wsl은 작업 하나를 받으면 사람 개입 없이 아래 순서를 스스로 끝까지 밟는다. 각 단계는 같은 이름의 스킬을 호출한다.

1. **unskein-wiki-search** — 작업을 시작하기 전에 이 repo에 이미 쌓인 지식(패턴·결정·주의사항)을 먼저 찾아 재분석을 막는다.
2. **unskein-scope** — 받은 작업을 이해하고, 검증 가능한 수용 기준으로 바꾼다.
3. **unskein-exec** — 정해진 범위를 최소·수술적으로 구현한다.
4. **unskein-verify** — 변경을 자체 검증한다. 스택을 감지해 타입체크·빌드·테스트를 돌리고 결과를 확인한다.
5. **unskein-wiki-ingest** — 작업에서 얻은 지식(패턴·교훈·결정)을 이 repo에 기록해 다음 작업이 재사용하게 한다.
6. **unskein-wiki-lint** — repo에 쌓인 지식 문서의 부패를 점검한다 — 깨진 링크, 오래된(stale) 내용, 중복·충돌.
7. **unskein-deploy** — 검증을 통과한 변경을 커밋·머지·push 하고, 배포 정보가 구성돼 있으면 그대로 배포한다. git 절차(머지·push·미러 점검·유출 방지)를 모두 포함한다.

마지막에 결과를 한 줄로 회수한다: 완료면 `RESULT: <요약>`, 사람 판단이 필요하면 `QUESTION: <질문>` (출력 규약은 `dao-skills/CLAUDE.md` 참조).

## 스킬별 상세

### 1. unskein-wiki-search

- **용도**: 작업을 시작하기 전에 이 repo에 이미 쌓인 지식(패턴·결정·주의사항)을 먼저 찾아 재분석을 막는다.
- **단계 위치**: 1 / 7
- **트리거 키워드**: 작업 시작, 기존 지식 검색, repo 파악, 무엇부터, 작업 전 조사.

### 2. unskein-scope

- **용도**: 받은 작업을 이해하고, 검증 가능한 수용 기준으로 바꾼다.

### 3. unskein-exec

- **용도**: 정해진 범위를 최소·수술적으로 구현한다.
- **단계 위치**: 3 / 7

### 4. unskein-verify
