# UnSkein — 사이트(프로젝트) 추가

모리 클라이언트가 새 프로젝트를 개발할 수 있도록, 그 프로젝트를 클라이언트에 등록하고 다오 실행 환경을 WSL에 구성한다. 클라이언트 주인(사용자)과 대화하며 아래 순서로 진행한다. 값이 빠지면 임의로 채우지 말고 사용자에게 물어 멈춘다.

## 1. WSL 준비 (윈도우에서 실행할 때)

다오(`claude -p`)는 WSL 안에서 구동한다 — 윈도우 콘솔 인코딩·도구 설치 마찰을 피한다.

- `wsl -l -v` 로 WSL 배포판이 있는지 확인한다. 없으면 `wsl --install` 설치를 안내(설치 후 재부팅 필요)하고 멈춘다.
- WSL 안에 `claude`(Claude Code)와 `git` 이 있는지 확인한다. 없으면 설치를 안내한다.
- 이미 WSL/리눅스 안에서 실행 중이면 이 단계는 건너뛴다.

## 2. 프로젝트 성격 파악

- 사용자에게 프로젝트 이름과 저장소 주소를 묻는다.
- 접근 가능하면 저장소를 얕게 살펴보고, 아니면 사용자에게 물어 언어·스택을 파악한다 (node / python / 기타).
- 다오가 그 프로젝트를 빌드·테스트하려면 필요한 런타임이 WSL에 있어야 한다. 없으면 설치를 안내한다.

## 3. git 정보·자격증명 받기

- 저장소 주소를 받고 전송 방식을 판정한다: `https://` → 토큰, `git@`/`ssh://` → SSH 개인키.
- 자격증명을 `UNSKEIN_CRED_DIR`(기본 `~/.unskein/creds`, WSL 경로)에 둔다:
  - HTTPS: 개인 액세스 토큰을 `creds/.env` 의 `UNSKEIN_GIT_TOKEN=` 로 저장한다. 호스트가 여럿이면 `UNSKEIN_GIT_TOKEN_<HOST>=` 로 구분한다.
  - SSH: 개인키를 `creds/id_ed25519`(또는 `id_rsa`)에 두고 권한을 `600` 으로 맞춘다.
- 토큰·키는 화면에 다시 출력하지 않는다. 저장소 주소나 git 설정에 토큰을 넣지 않는다.

## 4. 연결 환경변수 확인

- `UNSKEIN_API`(작업 큐 주소), `UNSKEIN_MORI_TOKEN`(연결 토큰)이 WSL 환경에 설정됐는지 확인한다. 없으면 UnSkein 설정 화면에서 발급·확인하도록 안내한다.
- 다오 작업 폴더 `UNSKEIN_WORK_ROOT`(기본 `~/.unskein/work`, WSL 경로)를 확인한다.

## 5. 검증

- 받은 자격증명으로 저장소 접근을 1회 확인한다: `git ls-remote <repo>`(또는 얕은 클론).
- 성공하면 "이 프로젝트로 작업 큐의 작업을 받을 준비 완료"를 알리고, 다음 단계로 `/unskein:mori-once` 또는 `/unskein:mori-loop` 실행을 안내한다.
- 실패하면 사유(인증·네트워크·주소 형식)를 그대로 보여주고 멈춘다. 다른 값으로 우회하지 않는다.
