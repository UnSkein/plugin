# 위키 검색 — 작업 전

새 작업을 분석하기 전에, 이 repo가 이미 답을 갖고 있는지 먼저 본다.

1. repo에서 다음 위치를 확인한다 (있는 것만):
   - `docs/architecture/` — 도메인 개념·패턴·주의사항(Gotcha)
   - `docs/decisions/` — 기술 결정 이력(ADR)
   - `CLAUDE.md` / `README.md` — 프로젝트 규칙·구조
2. 작업 키워드로 위 문서를 검색해, 관련된 기존 패턴·결정·주의사항을 읽는다.
3. 찾은 내용을 다음 단계(scope·exec)에 반영한다 — 이미 결정된 것을 다시 분석하지 않는다.

위 문서가 없으면 그대로 진행한다 (처음 작업하는 repo일 수 있다). 지식을 새로 남기는 일은 `unskein-wiki-record`에서 한다.
