# 구현

이 단계는 `exec` status로 claim된다. 이전 단계(`scope`)의 수용 기준을 모리가 "이전 단계 계획"으로 프롬프트에 전달한다. 그 기준을 만족하는 구현만 한다.

1. scope에서 정한 기준을 만족하는 최소 변경을 한다. 추측성 기능·과한 추상화를 더하지 않는다.
2. 기존 코드 스타일을 따른다. 무관한 코드·포맷을 건드리지 않는다.
3. 내 변경이 만든 미사용 import·변수만 정리한다. 기존에 있던 dead code는 손대지 않는다.
4. 막히면(진전 없음, 인증·의존성 누락 등) 우회하지 말고 `QUESTION:`으로 사유를 드러낸다.
5. **커밋·push 하지 않는다.** 작업트리 수정만 한다. 커밋·머지·push는 마감(`deploy`) 단계에서만 한다.

## 보고 — 산출 본문 없음

구현 단계는 별도 산출 본문이 없다. `RESULT` 마커 첫 줄만 내고 펜스 블록은 생략한다. 보고 next status는 `test`다.

```
RESULT: status=test stage=exec summary=<한 줄 요약>
```
