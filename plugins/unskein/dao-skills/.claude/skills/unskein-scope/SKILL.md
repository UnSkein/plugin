# 범위 — 작업 이해

1. 작업 프롬프트(제목·설명)와 위키 검색 결과를 합쳐, 무엇을 완료로 볼지 정한다.
2. 요구사항이 모호하면 임의로 정하지 말고 `QUESTION:`으로 되묻는다 (요구사항 추측 금지).
3. 성공 기준을 검증 가능한 형태로 적는다 — 입력/동작/출력, 또는 화면/상태 변화. 이 기준이 `unskein-verify`의 점검 대상이 된다.
4. 범위를 받은 작업에 직접 닿는 것으로 한정한다. 요청하지 않은 개선은 하지 않는다.
